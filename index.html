Coisas com as quais trabalhei

Filosofia

Acredito que o código deva falar por si mesmo. Com foco em componentes e interfaces limpas, gosto de usar princípios de programação funcional para resolver problemas. Aprecio coisas simples. Comidas simples, bebidas simples, código simples. Até os problemas mais complexos podem ser divididos em peças únicas, isoladas e legíveis, como uma receita. Código escrito assim é mais fácil de testar, manter e escalar.

Princípios

  • KISS (Keep It Simple, mantenha simplicidade)
  • DRY (Don't Repeat Yourself, não se repita)
  • Organização (Kanban, etc)
  • Paradigma funcional
